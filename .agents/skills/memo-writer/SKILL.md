# When to use
Use this skill when the task is to produce a memo, investment summary, thesis, or risk/catalyst analysis from evidence already retrieved by the application.

# Rules
1. Read the evidence first.
2. Separate facts from inference.
3. Never present unsupported claims as facts.
4. Prefer direct citation handles or chunk references when available.
5. If evidence is weak, explicitly say what is missing.

# Output structure
- Thesis
- Key KPIs
- Risks
- Catalysts
- Open Questions
- Confidence Level

# Quality bar
- Avoid generic finance filler.
- Keep each claim tied to evidence.
- If there is a contradiction in sources, call it out clearly.
